# 项目代码修改：支持 DeepSeek API

本文档指导学员在当前 Week4 项目基础上，逐步把原来的“模板式 RAG 回答”升级为“接入真实 DeepSeek 大模型生成回答”。

当前项目已经完成了：

```text
文档加载 -> 文本切分 -> TF-IDF 检索 -> 返回相关 chunk -> 模板拼接回答
```

本次修改的目标是把最后一步替换为：

```text
文档加载 -> 文本切分 -> TF-IDF 检索 -> 返回相关 chunk -> DeepSeek API 生成回答
```

也就是说，检索部分仍然沿用当前代码，大模型只负责根据检索到的上下文生成更自然、更完整的答案。

---

## 1. 修改前先理解当前代码

当前 RAG 问答入口是：

```powershell
python src/rag_ask.py --question "KiCad 的 PCB 设计流程是什么？"
```

核心代码在：

```text
src/rag_ask.py
```

当前 `rag_ask.py` 的主要函数是：

```text
load_index()
retrieve()
build_answer()
ask()
main()
```

它们的职责如下：

- `load_index()`：读取 `vector_store/rag_index.pkl`。
- `retrieve()`：根据用户问题检索 top-k 个相关 chunk。
- `build_answer()`：把检索结果用模板拼成答案。
- `ask()`：组织完整问答流程，并把结果保存到 `results/`。
- `main()`：处理命令行参数。

本次修改的重点是替换 `build_answer()`。

## 2. DeepSeek API 当前接入方式

DeepSeek API 官方文档地址：

```text
https://api-docs.deepseek.com/
```

DeepSeek API 支持 OpenAI 兼容格式。根据官方文档，OpenAI 格式的基础配置是：

```text
base_url = https://api.deepseek.com
api_key = DEEPSEEK_API_KEY
model = deepseek-v4-flash 或 deepseek-v4-pro
```

官方示例使用 OpenAI SDK：

```python
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)
```

注意：

- `deepseek-v4-flash`：适合大多数练习场景，成本更低、速度更快。
- `deepseek-v4-pro`：适合更高质量或更强推理需求。
- `deepseek-chat` 和 `deepseek-reasoner` 在官方文档中标注为将于 `2026-07-24 15:59 UTC` 废弃，因此新代码建议直接使用 `deepseek-v4-flash` 或 `deepseek-v4-pro`。

参考官方文档：

- DeepSeek API Quick Start: <https://api-docs.deepseek.com/>
- DeepSeek Models & Pricing: <https://api-docs.deepseek.com/quick_start/pricing>

## 8. 第六步：新增 deepseek_client.py

为了让代码结构清晰，建议新建一个独立文件：

```text
src/deepseek_client.py
```

这个文件专门负责调用 DeepSeek API。

写入：

```python
from openai import OpenAI

from config import DEEPSEEK_API_KEY, DEEPSEEK_BASE_URL, DEEPSEEK_MODEL


def get_deepseek_client():
    if not DEEPSEEK_API_KEY:
        raise RuntimeError(
            "DEEPSEEK_API_KEY is not set. "
            "Please create a .env file or set the environment variable."
        )

    return OpenAI(
        api_key=DEEPSEEK_API_KEY,
        base_url=DEEPSEEK_BASE_URL,
    )


def generate_answer_with_deepseek(question: str, contexts, model: str | None = None) -> str:
    client = get_deepseek_client()
    selected_model = model or DEEPSEEK_MODEL

    context_text = "\n\n".join(
        f"[{item['rank']}] source={item['source']} score={item['score']:.4f}\n{item['text']}"
        for item in contexts
    )

    system_prompt = (
        "你是一个严谨的课程助教，负责根据本地知识库内容回答问题。"
        "请只依据给定的参考资料回答。"
        "如果资料不足，请明确说明资料中没有足够信息。"
        "回答要清晰、分点、适合初学者理解。"
    )

    user_prompt = f"""用户问题：
{question}

参考资料：
{context_text}

请基于参考资料回答用户问题。"""

    response = client.chat.completions.create(
        model=selected_model,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        temperature=0.2,
        stream=False,
    )

    return response.choices[0].message.content
```

代码说明：

- `get_deepseek_client()`：创建 DeepSeek 客户端。
- `generate_answer_with_deepseek()`：把问题和检索结果发送给大模型。
- `temperature=0.2`：让回答更稳定，减少发挥。
- `stream=False`：先使用非流式输出，便于初学者理解。

## 9. 第七步：修改 rag_ask.py

打开：

```text
src/rag_ask.py
```

当前文件中有一个模板回答函数：

```python
def build_answer(question: str, contexts):
    context_text = "\n\n".join(
        f"[{item['rank']}] source={item['source']} score={item['score']:.4f}\n{item['text']}"
        for item in contexts
    )
    answer = f"""问题：{question}

基于知识库检索到的内容，参考答案如下：

{contexts[0]['text']}

参考来源：
{context_text}
"""
    return answer
```

可以把它替换为调用 DeepSeek。

建议修改后的 `rag_ask.py` 完整版本如下：

```python
import argparse
import pickle

import pandas as pd
from sklearn.metrics.pairwise import cosine_similarity

from config import INDEX_PATH, RESULTS_DIR, ensure_dirs
from deepseek_client import generate_answer_with_deepseek


def load_index():
    if not INDEX_PATH.exists():
        raise FileNotFoundError(
            f"Index not found: {INDEX_PATH}\n"
            "Please run: python src/build_index.py"
        )
    with INDEX_PATH.open("rb") as f:
        return pickle.load(f)


def retrieve(question: str, top_k: int = 3):
    payload = load_index()
    vectorizer = payload["vectorizer"]
    matrix = payload["matrix"]
    chunks = payload["chunks"]

    query_vec = vectorizer.transform([question])
    scores = cosine_similarity(query_vec, matrix).flatten()
    top_indices = scores.argsort()[::-1][:top_k]

    results = []
    for rank, idx in enumerate(top_indices, start=1):
        item = dict(chunks[idx])
        item["rank"] = rank
        item["score"] = float(scores[idx])
        results.append(item)
    return results


def format_sources(contexts):
    return "\n\n".join(
        f"[{item['rank']}] source={item['source']} score={item['score']:.4f}\n{item['text']}"
        for item in contexts
    )


def build_answer(question: str, contexts, use_llm: bool = True):
    if use_llm:
        llm_answer = generate_answer_with_deepseek(question, contexts)
    else:
        llm_answer = contexts[0]["text"]

    sources = format_sources(contexts)
    answer = f"""问题：{question}

基于知识库检索到的内容，参考答案如下：

{llm_answer}

参考来源：
{sources}
"""
    return answer


def ask(question: str, top_k: int = 3, use_llm: bool = True):
    ensure_dirs()
    contexts = retrieve(question, top_k=top_k)
    answer = build_answer(question, contexts, use_llm=use_llm)

    result_path = RESULTS_DIR / "last_rag_answer.md"
    result_path.write_text(answer, encoding="utf-8")

    rows = pd.DataFrame(contexts)
    rows.to_csv(RESULTS_DIR / "last_retrieval_results.csv", index=False, encoding="utf-8-sig")

    return answer


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--question", required=True, help="Question to ask the RAG system")
    parser.add_argument("--top-k", type=int, default=3)
    parser.add_argument(
        "--no-llm",
        action="store_true",
        help="Use the old template answer instead of DeepSeek API",
    )
    args = parser.parse_args()

    answer = ask(args.question, top_k=args.top_k, use_llm=not args.no_llm)
    print(answer)


if __name__ == "__main__":
    main()
```

这个版本保留了原来的检索逻辑，只增加了：

- `from deepseek_client import generate_answer_with_deepseek`
- `format_sources()`
- `build_answer(..., use_llm=True)`
- `ask(..., use_llm=True)`
- 命令行参数 `--no-llm`

这样做的好处是：

- 默认使用 DeepSeek API 生成答案。
- 如果暂时没有 API Key，可以加 `--no-llm` 回到原模板模式。

## 15. 常见错误与排查

### 15.1 ModuleNotFoundError: No module named 'openai'

说明还没有安装 OpenAI SDK。

解决：

```powershell
pip install openai
```

或者：

```powershell
pip install -r requirements.txt
```

### 15.2 DEEPSEEK_API_KEY is not set

说明程序没有读到 API Key。

检查：

```text
.env
```

是否在项目根目录，并且内容类似：

```text
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
```

还要检查 `config.py` 中是否调用了：

```python
load_dotenv(ROOT_DIR / ".env")
```

### 15.3 401 Unauthorized

通常表示 API Key 不正确或无效。

处理方式：

- 检查 API Key 是否复制完整。
- 确认没有多余空格。
- 到 DeepSeek 平台重新创建 API Key。

### 15.4 余额不足或计费失败

DeepSeek API 调用可能需要账户余额。

处理方式：

- 登录 DeepSeek 平台查看余额。
- 检查模型选择是否符合预期。
- 课堂练习建议优先使用 `deepseek-v4-flash`。

### 15.5 网络连接失败

如果出现连接超时或 DNS 错误，通常和网络环境有关。

处理方式：

- 检查能否访问 `https://api.deepseek.com`。
- 换一个网络环境。
- 稍后重试。

### 15.6 输出中文乱码

如果 PowerShell 中中文显示乱码，但 `results/last_rag_answer.md` 文件内容正常，可以先直接打开结果文件查看。

也可以尝试在 PowerShell 中运行：

```powershell
chcp 65001
```

然后重新运行 Python 命令。

## 16. 课堂讲解重点

完成修改后，可以引导学员观察：

### 16.1 RAG 中检索和生成是两件事

检索仍由本地代码完成：

```text
TF-IDF -> cosine similarity -> top_k chunks
```

生成由 DeepSeek 完成：

```text
question + contexts -> DeepSeek -> answer
```

### 16.2 大模型不是直接回答，而是基于上下文回答

我们没有把问题直接丢给 DeepSeek，而是先检索知识库，再把检索结果放进 prompt。

这就是 RAG 的核心：

```text
让大模型基于外部知识回答，而不是只依赖模型自身记忆。
```

### 16.3 参考来源仍然很重要

即使使用了大模型，也要保留：

```text
source
score
chunk text
```

这样可以追踪答案依据，方便排查回答是否可靠。

### 16.4 API Key 不能写死在代码里

错误示范：

```python
client = OpenAI(api_key="sk-xxx")
```

正确做法：

```python
api_key=os.environ.get("DEEPSEEK_API_KEY")
```

或者通过 `.env` 加载。

## 18. 可选升级：支持 reasoning 参数

DeepSeek 官方示例中展示了 thinking mode 和 `reasoning_effort`。

对于入门课程，建议先不用这些参数，保持调用简单：

```python
response = client.chat.completions.create(
    model=selected_model,
    messages=[...],
    temperature=0.2,
    stream=False,
)
```

如果后续想尝试更强推理，可以在确认当前模型支持后，再加入类似参数：

```python
response = client.chat.completions.create(
    model=selected_model,
    messages=[...],
    temperature=0.2,
    stream=False,
    reasoning_effort="high",
    extra_body={"thinking": {"type": "enabled"}},
)
```

注意：

- 不同模型、不同模式支持的参数可能变化。
- 课堂基础版建议先使用普通 chat completion。
- 高级参数应以 DeepSeek 官方文档为准。

## 21. 一句话总结

本次修改的核心不是重写整个项目，而是只替换 RAG 的“生成回答”环节：

```text
原来：检索结果 -> 模板拼接回答
```

这样既保留了当前项目清晰的 RAG 工程结构，又让学员体验到真实大模型在知识库问答中的作用。
