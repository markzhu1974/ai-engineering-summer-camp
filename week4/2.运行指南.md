# Week 4 工程运行指南

本指南适合第一次实践 RAG、Agent 和 MCP-like 工具调用的学员。请按顺序完成，不要跳步。

---

## 1. 这周为什么用 Python 程序，而不是 Jupyter Notebook

Week1 和 Week2 更适合 Notebook，因为重点是探索数据、查看图表、逐步观察模型训练。

Week4 更适合 Python 程序工程，因为本周内容更像真实应用：

- RAG 要读取文档、切分文本、建立索引、保存索引。
- 问答系统要从命令行接收问题并输出答案。
- Agent 要根据问题选择工具。
- MCP 的核心思想是“程序通过标准接口发现和调用工具”。

所以本周主入口是：

```text
src/build_index.py
src/rag_ask.py
src/agent_demo.py
src/mcp_like_demo.py
```

## 14. 常见问题

### 14.1 `ModuleNotFoundError`

说明依赖没有安装到当前环境。运行：

```powershell
conda activate ai-camp-week4
pip install -r requirements.txt
```

### 14.2 提示找不到索引

先运行：

```powershell
python src/build_index.py
```

### 14.3 RAG 回答不够智能

本工程为了离线可运行，使用的是 TF-IDF 检索和简单模板回答，不依赖大模型 API。

它的目标是让你理解 RAG 工程结构，不是追求最终回答质量。

后续可以改进为：

- 使用 Embedding 模型。
- 使用向量数据库。
- 接入真实 LLM API。
- 使用 LangChain 或 LlamaIndex。

### 14.4 PDF 解析效果不好

PDF 本身可能是扫描件或排版复杂，`pypdf` 只能解析可提取文本的 PDF。

如果 PDF 是扫描件，需要 OCR 工具。
