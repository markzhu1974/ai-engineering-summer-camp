# 常用提示词模板

把下面模板复制到 Opencode 后，根据当前任务替换方括号内容。

## 1. 只读分析

```text
请只读分析当前项目，不要创建、修改或删除任何文件。
我想了解：[你想了解的问题]
请指出相关文件，并用初学者能理解的方式解释。
```

## 2. 修改前计划

```text
请先不要修改文件。
我想实现：[功能描述]
请给出实施计划：
1. 需要修改哪些文件。
2. 每个文件为什么要改。
3. 需要新增或修改哪些测试。
4. 如何手动验证。
5. 这个修改有没有风险。
```

## 3. 测试优先新增功能

```text
请使用测试优先的方式实现：[功能描述]

要求：
1. 先新增或修改 pytest 测试。
2. 再实现生产代码。
3. 不引入新框架。
4. 修改后说明改动文件。
5. 告诉我应该运行哪些命令验证。
```

## 4. 分析报错

```text
请只分析下面的报错，先不要修改文件。
请说明：
1. 最可能的原因。
2. 应该先检查哪个文件。
3. 最小修复方案是什么。

报错如下：
[粘贴完整报错]
```

## 5. 修复 Bug

```text
请根据下面的失败测试修复 Bug。

要求：
1. 只做最小必要修改。
2. 不改变无关功能。
3. 修复后解释根因。
4. 告诉我重新运行哪些命令。

失败信息：
[粘贴完整报错]
```

## 6. 重构

```text
请只执行一次小范围重构。

目标：[重构目标]
要求：
1. 不新增功能。
2. 不改变外部行为。
3. 保持现有测试通过。
4. 修改后说明为什么行为没有变化。
```

## 7. 代码审查

```text
请以代码审查方式只读检查当前项目，不要修改文件。
请按严重程度列出问题：
1. 可能导致错误行为的问题。
2. 测试缺口。
3. 可维护性问题。
4. 文档缺口。
每个问题都要指出具体文件和原因。
```

## 8. README 生成

```text
请根据当前项目实际代码编写 README.md。
不要写没有实现的功能。
README 必须包括：
1. 项目简介。
2. 环境要求。
3. 测试命令。
4. 打包命令。
5. 启动命令。
6. 支持的命令。
7. 5 分钟试用流程。
```

## 9. AGENTS.md 规则

```text
请创建或更新 AGENTS.md，记录 AI 助手在本项目中必须遵守的规则。
规则应覆盖技术栈、测试要求、修改前计划、修改后验证、安全边界和禁止读取密钥。
```
