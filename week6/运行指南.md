# Week 6 运行指南：用 Opencode + DeepSeek 完成 Python Task Tracker

本指南适合 Windows + VS Code 环境。所有路径都使用相对路径，假设你已经进入 `week6` 目录。

## 0. 本周你会做出什么

你将从一个空的 `task-tracker` 文件夹开始，让 Opencode 连接 DeepSeek，逐步完成一个 Python 命令行待办事项程序。

程序最终应该支持：

- 新增任务
- 查看任务列表
- 标记任务完成
- 搜索任务
- 保存和读取本地 JSON 数据
- 有一组可运行的 pytest 测试
- 有 README 和项目规则文件

本周不使用 Git。每次大改之前，用复制文件夹的方式创建本地快照。

## 1. 安装 VS Code

1. 打开 VS Code 官网：https://code.visualstudio.com/
2. 点击 Download for Windows。
3. 下载完成后运行安装程序。
4. 安装时建议勾选：
   - Add to PATH
   - Open with Code
   - Register Code as an editor
5. 安装完成后打开 VS Code。
6. 在 VS Code 左侧点击 Extensions 图标。
7. 搜索并安装：
   - Python
   - Pylance
   - Jupyter
   - Markdown All in One

说明：本周不是 Notebook 课程，Jupyter 不是必须用于运行项目；安装它是为了方便以后查看 Python 代码、实验片段和数据分析内容。

## 2. 安装 Python

1. 打开 Python 官网：https://www.python.org/downloads/
2. 点击 Download Python。
3. 运行安装程序。
4. 安装第一页必须勾选：

```text
Add python.exe to PATH
```

5. 点击 Install Now。
6. 安装完成后，重新打开 PowerShell。
7. 运行：

```powershell
python --version
pip --version
```

8. 如果能看到 Python 版本号和 pip 版本号，说明安装成功。

如果 `python` 命令不可用，可以尝试：

```powershell
py --version
```

如果 `py` 可用，后续命令中的 `python` 可以替换为 `py`。

## 3. 安装 Node.js

Opencode 通常通过 Node.js/npm 安装或运行。安装前先准备 Node.js。

1. 打开 Node.js 官网：https://nodejs.org/
2. 下载 LTS 版本。
3. 运行安装程序，保持默认选项即可。
4. 重新打开 PowerShell，运行：

```powershell
node -v
npm -v
```

5. 如果两个命令都能显示版本号，说明 Node.js 安装成功。

## 4. 安装 Opencode

Opencode 的安装方式可能随版本变化，请以官方文档为准：https://opencode.ai/docs/

如果官方文档仍提供 npm 安装方式，可以在 PowerShell 中尝试：

```powershell
npm install -g opencode-ai
```

安装后运行：

```powershell
opencode --version
```

如果能显示版本号，说明安装成功。

如果命令不存在，回到 Opencode 官方文档，按最新的 Windows 安装方式重新安装。

## 5. 准备 DeepSeek API Key

1. 打开 DeepSeek API 文档：https://api-docs.deepseek.com/
2. 登录 DeepSeek 开放平台。
3. 进入 API Keys 页面。
4. 创建一个新的 API Key。
5. 复制后只保存到你自己的安全位置。

注意：

- 不要把 API Key 写进 Markdown、Python 代码、截图或聊天记录。
- 不要把 API Key 放进 `task-tracker` 项目文件。
- 如果某一步需要粘贴 API Key，只粘贴到 Opencode 的安全输入位置。

## 6. 在 VS Code 中打开 week6

1. 打开 VS Code。
2. 点击 File -> Open Folder。
3. 选择 `week6` 文件夹。
4. 打开 VS Code 终端：

```text
Terminal -> New Terminal
```

5. 在终端中确认当前位置：

```powershell
pwd
```

6. 如果你不在 `week6`，进入 `week6`：

```powershell
cd week6
```

如果你已经在课程根目录中，才需要这条命令。进入后再运行 `pwd` 确认。

## 7. 连接 Opencode 和 DeepSeek

1. 在 VS Code 终端中运行：

```powershell
opencode
```

2. 进入 Opencode 后，运行：

```text
/connect
```

3. 按界面提示选择 DeepSeek。
4. 粘贴你的 DeepSeek API Key。
5. 连接完成后，运行：

```text
/models
```

6. 选择 DeepSeek 模型。

模型名称以 Opencode 和 DeepSeek 当前界面显示为准。普通代码生成可以优先选择速度快、成本低的模型；复杂排错或重构再切换到推理能力更强的模型。

## 8. 创建练习项目文件夹

在 `week6` 目录下运行：

```powershell
mkdir task-tracker
mkdir snapshots
```

如果文件夹已经存在，PowerShell 会提示已存在，可以继续下一步。

进入项目文件夹：

```powershell
cd task-tracker
```

启动 Opencode：

```powershell
opencode
```

## 9. 创建 Python 虚拟环境

后续项目会放在 `task-tracker` 中。先在这个目录里创建虚拟环境：

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 提示脚本执行策略限制，先运行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

关闭当前终端，重新打开 VS Code 终端，再进入 `task-tracker` 并激活：

```powershell
cd task-tracker
.\.venv\Scripts\Activate.ps1
```

升级 pip：

```powershell
python -m pip install --upgrade pip
```

安装测试工具：

```powershell
python -m pip install pytest
```

## 10. 开始做实验

按这个顺序执行：

1. 先阅读 [docs/safety_rules.md](docs/safety_rules.md)。
2. 再打开 [docs/experiment_plan.md](docs/experiment_plan.md)。
3. 从“实验 0”开始，每次只做一个实验。
4. 每个实验都要完成：
   - 创建快照
   - 输入提示词
   - 查看 Opencode 修改
   - 手动运行命令验证
   - 记录结果

## 11. 常用命令

进入项目：

```powershell
cd task-tracker
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

运行测试：

```powershell
python -m pytest
```

运行程序：

```powershell
python -m task_tracker
```

回到 `week6`：

```powershell
cd ..
```

创建快照示例：

```powershell
Copy-Item -Recurse task-tracker snapshots/task-tracker-exp04-before
```

比较文件：

1. 在 VS Code 左侧资源管理器中找到快照文件。
2. 右键选择 Select for Compare。
3. 找到当前项目中的同名文件。
4. 右键选择 Compare with Selected。

## 12. 出问题时怎么办

如果 pytest 报错：

1. 复制完整报错。
2. 回到 Opencode。
3. 粘贴下面的提示词：

```text
请只分析下面的 pytest 报错，先不要修改文件。
请说明最可能的 3 个原因，并告诉我应该先检查哪个文件。

报错如下：
粘贴报错
```

如果 Opencode 改坏了项目：

1. 不要继续让它反复修改。
2. 回到 `week6`。
3. 从最近的快照复制一个恢复版本：

```powershell
Copy-Item -Recurse snapshots/task-tracker-exp04-before task-tracker-recovery
```

4. 用 VS Code 打开 `task-tracker-recovery`。
5. 激活虚拟环境并运行测试，确认恢复版本是否可用。

## 13. 最终验收

完成所有实验后，检查：

```powershell
cd task-tracker
.\.venv\Scripts\Activate.ps1
python -m pytest
python -m task_tracker
```

然后填写：

- `reports/prompt_log.md`
- `reports/bug_fix_log.md`
- `reports/final_report.md`

如果这些文件还不存在，从对应模板复制：

```powershell
Copy-Item reports/prompt_log_template.md reports/prompt_log.md
Copy-Item reports/bug_fix_log_template.md reports/bug_fix_log.md
Copy-Item reports/final_report_template.md reports/final_report.md
```

如果当前在 `task-tracker` 目录中，请先运行 `cd ..` 再复制模板。
