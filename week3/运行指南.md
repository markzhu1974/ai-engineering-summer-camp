# Week 3 工程运行指南

本指南适合第一次开发 LLM 应用的学员。请按顺序完成，不要跳步。

---

## 1. 你将完成什么

本工程会带你完成一个完整的 LLM 应用开发流程：

```text
安装工具
-> 打开 VS Code
-> 创建 Python 环境
-> 安装依赖
-> 安装 VS Code Jupyter 插件
-> 打开 Notebook
-> 选择 Python Kernel
-> 学习 Prompt Engineering
-> 体验 Hugging Face Pipeline
-> 配置或模拟 LLM API 调用
-> 构建带历史记录的聊天助手
-> 保存 Prompt 和对话记录
```

完成后，你会得到：

```text
prompts/prompt_examples.md
results/chat_history.json
results/api_response_example.json
results/prompt_template_result.md
```

---

## 8. 可选：配置真实 LLM API Key

本 Notebook 不强制要求 API Key。没有 API Key 时，会使用 mock 模式模拟 LLM 返回，保证完整流程可以运行。

如果你有 DeepSeek API Key：

1. 复制 `.env.example` 为 `.env`。
2. 在 `.env` 中填写：

```text
DEEPSEEK_API_KEY=你的_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

如果你使用其他 OpenAI 兼容 API，也可以填写：

```text
OPENAI_API_KEY=你的_api_key
OPENAI_BASE_URL=你的_base_url
OPENAI_MODEL=你的_model_name
```

注意：不要把 `.env` 上传到公开仓库。

## 12. 常见问题

### 12.1 VS Code 找不到 Python Kernel

先确认已经激活环境并安装 kernel：

```powershell
conda activate ai-camp-week3
python -m ipykernel install --user --name ai-camp-week3 --display-name "Python (ai-camp-week3)"
```

然后重启 VS Code。

### 12.2 运行时报 `ModuleNotFoundError`

说明依赖没有安装到当前环境。运行：

```powershell
conda activate ai-camp-week3
pip install -r requirements.txt
```

### 12.3 API 调用失败

可能原因：

- 没有配置 `.env`。
- API Key 填错。
- Base URL 填错。
- 当前网络无法访问 API 服务。

没有关系，Notebook 会自动使用 mock 模式继续运行。

### 12.4 Hugging Face 模型下载失败

Notebook 会捕获错误，并使用本地 mock 文本生成函数继续运行。下载失败不会影响本周核心流程。
