# Week 7 运行指南：项目优化与成果展示

本周主题是“从 Demo 到可展示项目”。你需要从前六周的成果中选择一个项目，把它整理成最终交付版本，并准备 15 分钟左右的项目展示。

所有路径都使用相对路径。假设你已经进入课程根目录，并能看到 `week1` 到 `week7` 这些文件夹。

## 0. 本周最终成果

完成本周后，你应该交付：

- 一个可以运行的最终项目：`week7/final_project`
- 一份清晰 README：`week7/final_project/README.md`
- 一份安装与运行说明：`week7/final_project/docs/setup.md`
- 一份演示脚本：`week7/demo/demo_script.md`
- 一份项目展示 PPT 提纲或 PPT 文件：`week7/slides/`
- 一份 AI Coding 记录：`week7/reports/ai_coding_log.md`
- 一份最终总结报告：`week7/reports/final_report.md`

本周不要求你重新做一个大项目。重点是把已有项目整理到“别人能看懂、能运行、能展示”的程度。

## 1. 安装和检查工具

### 1.1 检查 Python

打开 PowerShell：

```powershell
python --version
pip --version
```

如果没有 Python：

1. 打开 https://www.python.org/downloads/
2. 下载 Windows 版本。
3. 安装第一页勾选 `Add python.exe to PATH`。
4. 安装完成后重新打开 PowerShell，再运行上面的检查命令。

### 1.2 检查 VS Code

1. 打开 VS Code。
2. 点击左侧 Extensions。
3. 搜索并安装：
   - Python
   - Pylance
   - Markdown All in One
   - GitLens

VS Code 下载地址：https://code.visualstudio.com/

### 1.3 检查 Git

第七周会练习 GitHub 工程化，因此建议安装 Git。

```powershell
git --version
```

如果没有 Git：

1. 打开 https://git-scm.com/downloads
2. 下载 Windows 版本。
3. 按默认选项安装。
4. 重新打开 PowerShell。
5. 再运行：

```powershell
git --version
```

### 1.4 准备 GitHub 账号

如果你还没有 GitHub 账号：

1. 打开 https://github.com/
2. 点击 Sign up。
3. 注册并登录。

本指南会先完成本地 Git 仓库整理。上传 GitHub 是推荐步骤，如果网络或账号暂时不可用，可以先完成本地版本。

### 1.5 检查 Opencode 和 DeepSeek

如果你已经完成 Week 6，直接检查：

```powershell
opencode --version
```

如果 Opencode 不可用，请回到 `week6/运行指南.md`，完成 Opencode + DeepSeek 配置。

## 5. 第一次完整运行

### 5.1 Python 项目运行方式

如果项目是 Python 项目，先创建虚拟环境：

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

如果有 `requirements.txt`：

```powershell
python -m pip install -r requirements.txt
```

如果有 `pyproject.toml` 且 README 中要求安装当前项目：

```powershell
python -m pip install -e .
```

运行测试：

```powershell
python -m pytest
```

运行项目：

```powershell
python -m task_tracker
```

如果你的项目不是 `task_tracker`，请按项目 README 中的命令运行。

### 5.2 Week 4 项目运行方式

如果你选择的是 `week4` 项目，进入项目后运行：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python src/build_index.py
python src/rag_ask.py
python src/agent_demo.py
```

### 5.3 记录第一次运行结果

复制模板：

```powershell
cd ../..
Copy-Item reports/final_report_template.md reports/final_report.md
```

打开 `reports/final_report.md`，填写：

- 选择了哪个项目。
- 第一次运行是否成功。
- 失败时的完整报错。
- 你准备优先修复什么。

## 6. 用 Opencode + DeepSeek 做项目体检

进入最终项目目录，例如：

```powershell
cd final_project/task-tracker
```

启动 Opencode：

```powershell
opencode
```

输入：

```text
请只读分析当前项目，不要修改文件。
请帮我做一次最终交付前的项目体检，按下面结构输出：
1. 当前项目能解决什么问题。
2. 项目入口在哪里。
3. 安装和运行步骤是否清晰。
4. 代码结构是否清晰。
5. 测试是否足够。
6. README 还缺什么。
7. 最优先修复的 5 个问题。
```

把 Opencode 输出复制到：

```text
reports/ai_coding_log.md
```

如果文件不存在，先回到 `week7` 复制模板：

```powershell
cd ../..
Copy-Item reports/ai_coding_log_template.md reports/ai_coding_log.md
```

## 7. Day 1：功能完善

目标：项目至少能完整运行一次。

### 7.1 建立功能清单

在最终项目目录中创建或打开：

```text
docs/feature_checklist.md
```

如果 `docs` 不存在：

```powershell
mkdir docs
```

在文件中写：

```markdown
# Feature Checklist

- [ ] 功能 1：
- [ ] 功能 2：
- [ ] 功能 3：
- [ ] 安装流程可执行
- [ ] 运行流程可执行
- [ ] 有测试或手动验收步骤
```

### 7.2 让 AI 帮你补齐检查表

在 Opencode 中输入：

```text
请只读分析当前项目，不要修改文件。
请根据实际代码，为 docs/feature_checklist.md 提供一份完整的功能检查清单。
请区分“已经实现”“可能实现但需要验证”“没有实现”。
```

确认内容后，再让 Opencode 写入文件：

```text
请把刚才的功能检查清单写入 docs/feature_checklist.md。
```

### 7.3 修复一个最小问题

选择一个最小且明确的问题，例如：

- README 中运行命令错误。
- 缺少 requirements.txt。
- 某个测试失败。
- 程序启动后没有说明如何操作。

在 Opencode 中输入：

```text
请先不要修改文件。
我想修复这个问题：[粘贴问题]
请给出最小修改计划，包括要改哪些文件、为什么改、如何验证。
```

确认计划后再输入：

```text
按这个最小计划开始修改。修改后请告诉我验证命令。
```

运行验证命令，并把结果记录到 `reports/ai_coding_log.md`。

## 8. Day 2：AI Coding 重构

目标：不是重写项目，而是让项目更清楚、更稳定。

### 8.1 创建重构前快照

回到 `week7`：

```powershell
cd ../..
mkdir snapshots
Copy-Item -Recurse final_project snapshots/final_project-before-refactor
```

### 8.2 让 AI 提出重构建议

进入项目：

```powershell
cd final_project/task-tracker
```

在 Opencode 中输入：

```text
请只读分析当前项目，不要修改文件。
请提出 5 个小范围重构建议。
要求：
1. 每个建议都说明收益。
2. 每个建议都说明风险。
3. 每个建议都说明验证方式。
4. 不要建议重写整个项目。
```

### 8.3 只执行一个重构

选择最小的一个建议，然后输入：

```text
请只执行第 [编号] 个重构建议。
要求：
1. 不改变已有功能。
2. 不新增功能。
3. 保持测试通过。
4. 修改后列出改动文件和验证命令。
```

运行验证：

```powershell
python -m pytest
```

如果项目没有测试，至少运行项目主入口，并把手动操作步骤写入 `docs/manual_test.md`。

## 9. Day 3：GitHub 工程化

目标：项目结构清楚，提交历史清楚，README 能让别人运行。

### 9.1 初始化本地 Git 仓库

先确认你在哪里：

```powershell
pwd
```

如果你已经在最终项目目录，例如 `final_project/task-tracker`，可以直接继续。

如果你在 `week7` 目录下，进入最终项目：

```powershell
cd final_project/task-tracker
```

如果你在别的目录，先回到 `week7`，再进入最终项目。

初始化：

```powershell
git init
```

设置本项目用户名和邮箱：

```powershell
git config user.name "你的名字"
git config user.email "你的邮箱"
```

查看状态：

```powershell
git status
```

### 9.2 创建 .gitignore

如果没有 `.gitignore`，创建一个：

```powershell
New-Item .gitignore
```

然后在 VS Code 中打开 `.gitignore`，写入下面内容并保存：

```text
.venv/
__pycache__/
.pytest_cache/
*.pyc
.env
data/raw/
models/*.pkl
models/*.pt
```

如果你不确定哪些文件不应该提交，让 Opencode 只读分析：

```text
请只读分析当前项目，不要修改文件。
请告诉我这个 Python 项目应该在 .gitignore 中忽略哪些文件，并说明原因。
```

### 9.3 第一次提交

查看状态：

```powershell
git status
```

添加文件：

```powershell
git add .
```

提交：

```powershell
git commit -m "Initial final project version"
```

### 9.4 可选：上传 GitHub

1. 打开 https://github.com/
2. 点击右上角 `+`。
3. 选择 New repository。
4. Repository name 填你的项目名，例如 `ai-engineering-final-project`。
5. 先不要勾选添加 README、.gitignore 或 License。
6. 创建仓库后，GitHub 页面会显示本地仓库推送命令。
7. 在 PowerShell 中按页面提示执行。

如果你暂时不上传 GitHub，也要保留本地 Git 提交。
