# Week 2 工程运行指南

本指南适合第一次使用 PyTorch 做图像分类项目的学员。请按顺序完成，不要跳步。

---

## 1. 你将完成什么

本工程会带你完成一个完整的 PyTorch 图像分类项目：

```text
安装工具
-> 打开 VS Code
-> 创建 Python 环境
-> 安装 PyTorch 等依赖
-> 安装 VS Code Jupyter 插件
-> 打开 Notebook
-> 选择 Python Kernel
-> 从头运行 Notebook
-> 训练 MLP 图像分类模型
-> 保存模型、曲线、预测结果和评估指标
```

完成后，你会得到：

```text
figures/sample_images.png
figures/training_curves.png
figures/prediction_examples.png
models/mnist_mlp.pt
results/metrics.csv
```

## 2. 安装 Python 环境管理工具：Miniconda

## 6. 创建 Python 环境

回到 PowerShell，确认当前目录仍然是 `week2`。

```text
.../week2
```

创建环境：

```powershell
conda create -n ai-camp-week2 python=3.10 -y
```

激活环境：

```powershell
conda activate ai-camp-week2
```

检查 Python 版本：

```powershell
python --version
```

预期输出：

```text
Python 3.10.x
```

---

## 7. 安装项目依赖

确认已经激活环境：

```text
(ai-camp-week2)
```

然后运行：

```powershell
pip install -r requirements.txt
```

如果安装 PyTorch 很慢，可以先等待；PyTorch 包比较大，这是正常现象。

安装完成后，运行：

```powershell
python -m ipykernel install --user --name ai-camp-week2 --display-name "Python (ai-camp-week2)"
```

这一步会让 VS Code Notebook 能看到这个 Python 环境。

## 11. 常见问题

### 11.1 VS Code 找不到 Python Kernel

先确认已经激活环境并安装 kernel：

```powershell
conda activate ai-camp-week2
python -m ipykernel install --user --name ai-camp-week2 --display-name "Python (ai-camp-week2)"
```

然后重启 VS Code。

### 11.2 运行时报 `ModuleNotFoundError`

说明依赖没有安装到当前环境。运行：

```powershell
conda activate ai-camp-week2
pip install -r requirements.txt
```

### 11.3 MNIST 下载失败

Notebook 已经内置备用数据逻辑。MNIST 下载失败时，会自动使用 scikit-learn 的 digits 数据集继续训练流程。

### 11.4 训练速度慢

本项目默认只训练少量 epoch，CPU 也可以完成。如果机器较慢，可以在 Notebook 中把：

```python
EPOCHS = 3
```

改成：

```python
EPOCHS = 1
```

### 11.5 没有 GPU

没有 GPU 也可以运行。本项目会自动选择：

```text
cuda 或 cpu
```

如果没有 GPU，会使用 CPU。
