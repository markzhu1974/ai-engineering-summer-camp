# Week 1 工程运行指南

本指南适合第一次接触 AI 工程实践的学员。请按顺序完成，不要跳步。

---

## 1. 你将完成什么

本工程会带你完成一个完整的 AI 项目：

```text
安装工具
-> 打开 VS Code
-> 创建 Python 环境
-> 安装依赖
-> 安装 VS Code Jupyter 插件
-> 打开 Notebook
-> 选择 Python Kernel
-> 从头运行 Notebook
-> 得到数据、图表、模型和评估结果
```

完成后，你会得到：

```text
data/raw_dataset.csv
data/clean_dataset.csv
figures/feature_distribution.png
figures/correlation_heatmap.png
figures/prediction_vs_ground_truth.png
figures/residual_plot.png
models/linear_regression.pkl
results/metrics.csv
```

---

## 2. 安装 Python 环境管理工具：Miniconda

推荐使用 Miniconda 管理 Python 环境。

下载地址：

```text
https://docs.conda.io/en/latest/miniconda.html
```

安装时保持默认选项即可。

安装完成后，重新打开 PowerShell，运行：

```powershell
conda --version
```

如果看到类似下面的输出，说明安装成功：

```text
conda 24.x.x
```

如果提示 `conda` 不是命令，请关闭 PowerShell 后重新打开。如果仍然失败，重新安装 Miniconda。

---

## 3. 安装 VS Code

下载地址：

```text
https://code.visualstudio.com/Download
```

安装完成后，打开 PowerShell，运行：

```powershell
code --version
```

如果能看到版本号，说明安装成功。

如果提示 `code` 不是命令：

1. 打开 VS Code。
2. 按 `Ctrl + Shift + P`。
3. 搜索 `Shell Command: Install 'code' command in PATH`。
4. 执行该命令。
5. 关闭 PowerShell，重新打开后再运行 `code --version`。

---

## 4. 用 VS Code 打开 week1 工程

打开 PowerShell，先进入你保存训练营资料的目录。

例如：如果你把训练营资料放在桌面，就先进入桌面上的训练营目录；如果你把资料解压到了其他位置，就进入你自己的目录。

然后进入第一周工程目录：

```powershell
cd week1
```

用 VS Code 打开当前目录：

```powershell
code .
```

VS Code 打开后，左侧文件树应能看到：

```text
README.md
运行指南.md
requirements.txt
data/
figures/
models/
notebooks/
results/
```

---

## 5. 安装 VS Code 插件

在 VS Code 中：

1. 点击左侧 Extensions 图标。
2. 或按 `Ctrl + Shift + X`。
3. 搜索并安装 `Python` 插件，发布者为 Microsoft。
4. 搜索并安装 `Jupyter` 插件，发布者为 Microsoft。

如果 VS Code 右下角提示安装推荐插件，也可以直接点击安装。

---

## 6. 创建 Python 环境

回到 PowerShell，确认当前目录仍然是 `week1`。

```text
.../week1
```

创建环境：

```powershell
conda create -n ai-camp-week1 python=3.10 -y
```

激活环境：

```powershell
conda activate ai-camp-week1
```

检查 Python 版本：

```powershell
python --version
```

预期输出：

```text
Python 3.10.x
```

---

## 7. 安装项目依赖

确认已经激活环境：

```text
(ai-camp-week1)
```

然后运行：

```powershell
pip install -r requirements.txt
```

安装完成后，运行：

```powershell
python -m ipykernel install --user --name ai-camp-week1 --display-name "Python (ai-camp-week1)"
```

这一步会让 VS Code Notebook 能看到这个 Python 环境。

---

## 8. 打开 Notebook

在 VS Code 左侧文件树中打开：

```text
notebooks/week01_house_price_prediction.ipynb
```

打开后，右上角会看到 `Select Kernel` 或 Python 环境名称。

点击它，选择：

```text
Python (ai-camp-week1)
```

如果没有看到这个选项：

1. 确认第 7 步已经运行。
2. 关闭并重新打开 VS Code。
3. 再次打开 Notebook，重新选择 Kernel。

---

## 9. 运行 Notebook

推荐方式：

1. 打开 Notebook。
2. 点击顶部 `Run All`。
3. 从上到下观察每个 cell 的输出。

也可以逐个 cell 运行：

1. 鼠标点击第一个 cell。
2. 点击 cell 左侧的运行按钮。
3. 等待输出出现。
4. 继续运行下一个 cell。

第一次学习建议逐个 cell 运行，因为这样更容易理解每一步发生了什么。

## 11. 常见问题

### 11.1 VS Code 找不到 Python Kernel

先确认已经激活环境并安装 kernel：

```powershell
conda activate ai-camp-week1
python -m ipykernel install --user --name ai-camp-week1 --display-name "Python (ai-camp-week1)"
```

然后重启 VS Code。

### 11.2 运行时报 `ModuleNotFoundError`

说明依赖没有安装到当前环境。运行：

```powershell
conda activate ai-camp-week1
pip install -r requirements.txt
```

### 11.3 下载 California Housing 数据失败

Notebook 已经内置离线备用数据生成逻辑。下载失败时，会自动生成一个学生成绩预测数据集，并继续完成后续流程。

### 11.4 图片没有生成

确认 Notebook 已经从头运行到最后。

也可以检查 `figures/` 目录：

```powershell
dir figures
```

### 11.5 模型文件没有生成

确认 Notebook 中“保存模型”部分已经运行。

检查：

```powershell
dir models
```
